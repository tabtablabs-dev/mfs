# v0.0.1 Release Notes

`mfs` v0.0.1 is the first OSS alpha release: a read-only Modal Volume filesystem/query CLI for agents.

## What ships

```text
mfs version [--json]
mfs doctor [TARGET] [--json]
mfs ls TARGET [--limit N] [--recursive] [--json]
mfs stat TARGET [--json]
mfs cat TARGET [--bytes START:LEN] [--max-bytes N] [--json]
```

## Why this release is intentionally small

The spec originally aimed at the full agent-filesystem surface: index, grep, search, manifests, change detection, and guarded writes. The Modal profile/Volume spike showed two constraints that should shape the real implementation before adding broad features:

- Modal path listings can fail as too broad even with `max_entries`.
- Public Modal SDK methods do not expose all bounded semantics needed for safe agent use.

So v0.0.1 ships the reliable core first: explicit addressing, JSON contracts, profile-safe Modal access, bounded listing, bounded stat, and bounded cat.

## Verified live behavior

The release was smoke-tested against a real `tabtablabs/main` Modal profile/environment:

- `mfs doctor ... --json`
- `mfs ls Volumes/ --json`
- `mfs ls Volumes/modal --json`
- `mfs ls Volumes/modal/tabtablabs/main --limit 1 --json`
- `mfs ls Volumes/modal/tabtablabs/main/<volume> --limit 3 --json`
- `mfs stat Volumes/modal/tabtablabs/main/<volume>/campaign.json --json`
- `mfs cat Volumes/modal/tabtablabs/main/<volume>/campaign.json --bytes 0:64 --json`

## Known limitations

- Alpha API; JSON shape may still evolve before v0.1.0.
- Uses Modal private/proto internals to preserve bounded reads/listings.
- Read-only: no upload/download/write/delete/copy commands yet.
- No sidecar SQLite index yet.
- No PyPI or GitHub release has been published by this local release prep.
